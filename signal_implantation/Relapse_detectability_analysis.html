1 Overview

This document is a step-by-step walkthrough for reproducing the breast cancer recurrence biomarker detectability and calibration analysis in the manuscript. The analysis starts from a single phyloseq object and proceeds through four main stages:

  1. prepare the count table and sample metadata;
  2. generate signal-implantation simulations with a patched version of SIMBA;
  3. benchmark multiple differential abundance methods on the simulated data; and
  4. compare the real-data results with simulation-derived detectability boundaries.

The goal is not only to reproduce the figures in the manuscript, but also to make the logic of the workflow transparent to anyone who wants to rerun or adapt it to a comparable dataset.

2 Before you start

2.1 Required inputs

To reproduce the analysis, you will need:

  • a phyloseq object containing the taxon count table and sample metadata;
  • installation of the patched SIMBA fork from https://github.com/microbiomix/SIMBA
    • (use devtools::install_github(repo="https://github.com/microbiomix/SIMBA"))
  • the companion helper scripts helper_functions.R, helper_detectability.R; and
  • a writable temporary directory for the HDF5 simulation file and output figures.

2.2 What this document assumes

This workflow assumes that the starting phyloseq object contains:

  • a count-based OTU table;
  • a sample-level identifier that can be used as row names for metadata; and
  • the clinical variables needed for the benchmarked models.

In the present breast cancer analysis, the main outcome is Relapse.binary, and the covariate-adjusted benchmark adjusts for ki67_ihc, bmi_calculated, ln_tumorpositive.binary.

5 Load and prepare the phyloseq object

5.1 Global analysis settings

These settings define the minimum prevalence filter used throughout the analysis and the significance threshold used in the main benchmark summaries. They also define the number of reps made in the simulation, and how many of them are used for benchmarking.

6 Define simulation parameters

We now specify the filtering parameters used internally by SIMBA, the proportion of implanted marker taxa, class balance, subset size, the abundance-shift grid and the number of simulation repeats.

In this benchmark, ab.scale is the primary implanted effect-size parameter. We set prev.scale = 0, meaning that signals are implanted as abundance differences without additional prevalence shifts.

# SIMBA filtering parameters stored in the HDF5 file
ab.cutoff <- 1e-04
prev.cutoff <- MIN_BENCHMARK_PREVALENCE
log.n0 <- 1e-05

# Simulation design
prop.markers <- 0.1
class.balance <- 15 / 99
subset_size <- 99
sim_reps <- REPS_MADE

# Implanted effect sizes
ab.scale <- seq(1, 10, 0.5)
prev.scale <- 0

7 Generate the signal-implantation simulations

The patched SIMBA workflow preserves the original sample-to-metadata mapping and allows covariates to remain attached to the same samples during benchmarking. This is crucial when the downstream methods use covariate adjustment.

7.1 Create the simulation HDF5 file

Because labels are fixed in this benchmark, we only need one testing-index repetition for each subset size.

      feat = feat,
      meta = meta,
      sim.location = sim.implant,
      sim.type = "cross-section",
      sim.method = "resampling",

        ab.cutoff = as.numeric(ab.cutoff),

9 Prepare the benchmark groups

SIMBA stores one HDF5 group per simulation setting. The chunks below identify all groups and restrict the analysis to the intended number of simulation repeats.

temp <- rhdf5::h5ls(sim.implant, recursive = FALSE)
groups <- setdiff(temp$name, c("original_data", "simulation_parameters"))

# Keep abundance-scale groups with the single prevalence setting used here.
groups.sub <- groups[grepl("^ab[0-9]+_prev1_rep[0-9]+$", groups)]

# Restrict to the requested number of simulation repeats.
groups.sub <- groups.sub[
  as.integer(stringr::str_match(groups.sub, "^ab[0-9]+_prev1_rep([0-9]+)$")[, 2]) <= REPS_USED
]

temp <- rhdf5::h5ls(sim.null, recursive = FALSE)
groups <- setdiff(temp$name, c("original_data", "simulation_parameters"))

# Keep abundance-scale groups with the single prevalence setting used here.
groups.sub.null <- groups[grepl("^ab[0-9]+_prev1_rep[0-9]+$", groups)]

rm(temp, groups)

10 Benchmark multiple differential abundance methods

The benchmark runs one method at a time across all simulated groups and then aggregates performance metrics across repeats.

10.1 Run the implantation-based benchmark

The normalization strategy and covariate adjustment are chosen separately for each method. Calibration tables are also extracted where available, as they are needed later for detectability-boundary analyses.

test_names <- c(
  "LinDA",
  "metagenomeSeq",
  "ANCOMBC2", "ANCOMBC",
  "DESeq2", "lm", "edgeR"
)

force_DA_tests <- FALSE
filename = file.path(simh5.loc, paste0("recurrence_", param_set, "_eval_summary.rds"))
if (!file.exists(filename) || force_DA_tests) {
  
  # Modify this 
  n_workers <- n_cpus
  
  if (n_workers > 1) {
    future::plan(future::multisession, workers = n_workers)
  } else {
    future::plan("sequential")
  }
  
  method_results_recurrence <- test_names %>%
    set_names() %>%
    purrr::map(function(test_name) {
      # Method-specific normalization
      norm_method <- dplyr::case_when(
        test_name == "lm" ~ "clr_halfmin",
        test_name == "wilcoxon" ~ "TSS",
        test_name == "LinDA" ~ "TSS",
        TRUE ~ "pass"
      )
      
      # Covariate adjustment where relevant
      covariates <- NULL
      if (test_name %in% c("LinDA", "metagenomeSeq", "lm", "ANCOMBC", "ANCOMBC2", "DESeq2", "edgeR")) {
        covariates <- c("bmi_calculated", "ki67_ihc", "ln_tumorpositive.binary")
      }
      
      # Some methods do not currently return effect sizes in the patched workflow
      return_calib <- TRUE
      
      simbaR::run_method_eval_parallel(
        test_name = test_name,
        groups.sub = groups.sub,
        sim.implant = sim.implant,
        subset_size = subset_size,
        ab.scale = ab.scale,
        alpha = ALPHA,
        norm = norm_method,
        covariates = covariates,
        seed = TRUE,
        show_progress = TRUE,
        simba.loc = NULL,
        return_calib = return_calib
      )
    })
  saveRDS(method_results_recurrence, file = filename)
} else {
  method_results_recurrence <- readRDS(file = filename)
}

10.2 Summarize benchmark performance

master_evals_recurrence <- dplyr::bind_rows(purrr::map(method_results_recurrence, "evals"))

power_combined_benchmark_recurrence <- master_evals_recurrence %>%
  dplyr::filter(test %in% test_names) %>%
  group_by(test, ab_scale, prev_index) %>%

    sd_power = sd(power, na.rm = TRUE),
    mean_fdr = mean(empirical_fdr, na.rm = TRUE),

    sd_abs_eff_tp = sd(mean_abs_eff_tp, na.rm = TRUE),
    mean_abs_eff_marker2 = mean(mean_abs_eff_marker, na.rm = TRUE),
    sd_abs_eff_marker = sd(mean_abs_eff_marker, na.rm = TRUE),

11 Select methods for interpretability analysis

Only methods that jointly satisfy the chosen benchmark criteria are carried forward into the detectability-boundary plots. We have relaxed the criteria to be: above 40% power and below 15% FDR.

13 Run the real-data differential abundance analyses

The next step is to run the corresponding differential abundance methods on the real breast cancer case/control data. These results are then compared with the simulation-derived detectability boundaries.

# prepare the tables
ps2 <- filter_taxa_by_prevalence(ps, MIN_DISCOVERY_PREVALENCE)
ps2rel <- transform_sample_counts(ps2, function(x) x / sum(x))
real_feat_relab <- as(otu_table(ps2rel), "matrix")
real_feat_relab <- as.data.frame(real_feat_relab[, rownames(meta)])

force_DA_tests <- FALSE

param_set2 <- paste("alpha", ALPHA, "prev", MIN_DISCOVERY_PREVALENCE, "reps", REPS_USED, sep = "_")
filename = file.path(simh5.loc, paste0("recurrence_", param_set2, "_real_data.rds"))

if (!file.exists(filename) || force_DA_tests) {
  
  # Encode from 1/2 to 0/1
  
  meta$relapse.binary <- meta$relapse.binary - 1
  sample_data(ps2) <- meta
  
  # ANCOMBC2
  res_ancombc2 <- ancombc2(
    data = ps2,
    fix_formula = "relapse.binary + bmi_calculated + ki67_ihc + ln_tumorpositive.binary",
    rand_formula = NULL,
    p_adj_method = "fdr",
    prv_cut = MIN_DISCOVERY_PREVALENCE,
    lib_cut = 1000,
    alpha = ALPHA
  )
  # Filter for sensitivity analysis
  res_ancombc2$res <- res_ancombc2$res %>%
    dplyr::mutate(p_relapse.binary = replace(p_relapse.binary, passed_ss_relapse.binary != TRUE, 1))
  
  # ANCOMBC
  res_ancombc <- ancombc(
    data = ps2,
    formula = "relapse.binary + bmi_calculated + ki67_ihc + ln_tumorpositive.binary",
    p_adj_method = "fdr",
    pseudo = 1,
    prv_cut = MIN_DISCOVERY_PREVALENCE,
    lib_cut = 1000
  )
  
  # Re-encode from 0/1 to -1/+1
  newlabel <- as.numeric(meta$relapse.binary)
  newlabel[newlabel == 0] <- -1
  # newlabel[newlabel == 1] <- 1
  meta$relapse.binary <- newlabel
  
  # lm on clr-transformed abundance with a half-minimum pseudocount
  tab_analysis <- as(otu_table(ps2), "matrix")
  pseudo <- min(tab_analysis[tab_analysis > 0]) / 2
  tab_analysis[tab_analysis == 0] <- pseudo
  tab_analysis <- as.data.frame(apply(tab_analysis, 1, compositions::clr))
  
  fit_one <- function(y, meta) {
    d <- cbind(meta, y = y)
    fit <- stats::lm(y ~ relapse.binary + bmi_calculated + ki67_ihc + ln_tumorpositive.binary, data = d)
    res <- broom.mixed::tidy(fit, effects = "fixed")
    res[res$term == "relapse.binary", ]
  }
  
  res_lm <- apply(tab_analysis, MARGIN=2, FUN = fit_one, meta = meta) %>%
    bind_rows(.id = "OTU") %>%
    mutate(padj = p.adjust(p.value, method = "fdr"))
  
  # LinDA on relative abundance
  
  meta$relapse.binary = factor(meta$relapse.binary, levels=c(-1, 1))
  res_linda <- LinDA::linda(
    real_feat_relab,
    meta,
    formula = "~ relapse.binary + bmi_calculated + ki67_ihc + ln_tumorpositive.binary",
    alpha = ALPHA,
    p.adj.method = "BH",
    type = "proportion"
  )
  
  # metagenomeSeq
  
  sample_data(ps2) <- meta # take the version with factors and -1/+1
  
  m <- phyloseq_to_metagenomeSeq(ps2)
  m2 <- filterData(m, present = nsamples(ps2) * MIN_DISCOVERY_PREVALENCE, depth = 1000)
  p <- cumNormStat(m2)
  m2 <- cumNorm(m2, p = p)
  pd <- pData(m2)
  
  mod1 <- model.matrix(~ relapse.binary + bmi_calculated + ki67_ihc + ln_tumorpositive.binary, data = pd)
  res1 <- fitZig(m2, mod1, control = zigControl(maxit = 20, verbose = FALSE))
  res_metagenomeSeq <- MRcoefs(res1, number = nrow(res1), adjustMethod = "fdr")
  
  # Store raw p-values and effect sizes for downstream interpretability analyses
  p_lm_real <- as.numeric(res_lm$p.value)
  eff_lm_real <- as.numeric(res_lm$estimate)
  p_linda_real = res_linda$output$relapse.binary$pvalue
  eff_linda_real = res_linda$output$relapse.binary$log2FoldChange
  p_mgs_real <- res_metagenomeSeq$pvalues
  eff_mgs_real <- res_metagenomeSeq$relapse.binary1
  p_ancombc_real <- res_ancombc$res$p_val$relapse.binary
  eff_ancombc_real <- res_ancombc$res$lfc$relapse.binary
  p_ancombc2_real <- res_ancombc2$res$p_relapse.binary
  eff_ancombc2_real <- res_ancombc2$res$lfc_relapse.binary
  
  names(p_lm_real) <- rownames(real_feat_relab)
  names(eff_lm_real) <- rownames(real_feat_relab)
  names(p_linda_real) <- rownames(real_feat_relab)
  names(eff_linda_real) <- rownames(real_feat_relab)
  names(p_mgs_real) <- rownames(real_feat_relab)
  names(eff_mgs_real) <- rownames(real_feat_relab)
  names(p_ancombc_real) <- rownames(real_feat_relab)
  names(eff_ancombc_real) <- rownames(real_feat_relab)
  names(p_ancombc2_real) <- res_ancombc2$res$taxon
  names(eff_ancombc2_real) <- res_ancombc2$res$taxon
  
  real_results_recurrence <- list(
    lm = list(pval = p_lm_real, eff = eff_lm_real),
    LinDA = list(pval = p_linda_real, eff = eff_linda_real),
    ANCOMBC = list(pval = p_ancombc_real, eff = eff_ancombc_real),
    ANCOMBC2 = list(pval = p_ancombc2_real, eff = eff_ancombc2_real),
    metagenomeSeq = list(pval = p_mgs_real, eff = eff_mgs_real)
  )
  
  saveRDS(real_results_recurrence, file = filename)
} else {
  real_results_recurrence <- readRDS(file = filename)
}
